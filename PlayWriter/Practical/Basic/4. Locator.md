# 🎯 Playwright Locators 
---

## 1. WHAT

**Locators** are used to identify elements on a web page.

They answer:

```text
Which element should Playwright interact with?
```

---

## 2. WHY

Bad locators = **flaky tests**

Problems:

* UI change → test breaks ❌
* dynamic classes → unstable ❌
* wrong element → false results ❌

Good locators:

* stable automation ✅
* readable tests ✅
* low maintenance ✅

---

## 3. WHEN TO USE

Locators are used:

* before every action (click, fill, select)
* before every assertion
* when extracting UI data
* when validating behavior

---

## 4. HOW

Execution flow:

```text
Locator → Auto-wait → Action → Assertion
```

---

## 5. REAL-LIFE ANALOGY

## 6. LOCATOR TYPES (PRIORITY ORDER)

---

## ✅ 1. By Role (BEST)

```ts
page.getByRole('button', { name: 'Login' });
```

✔ uses accessibility
✔ most stable
✔ preferred in enterprise

---

## ✅ 2. By Label

```ts
page.getByLabel('Username');
```

✔ ideal for forms

---

## ✅ 3. By Test ID (ENTERPRISE STANDARD)

```ts
page.getByTestId('login-btn');
```

Requires:

```html
<button data-testid="login-btn">Login</button>
```

✔ most reliable in real systems

---

## ✅ 4. By Text

```ts
page.getByText('Products');
```

⚠️ text may change

---

## ⚠️ 5. CSS Selector

```ts
page.locator('.btn.primary');
```

⚠️ can break easily

---

## ❌ 6. XPath (AVOID)

```ts
page.locator('//div[2]/button');
```

❌ brittle
❌ unreadable

---

## 7. ENTERPRISE PRIORITY RULE

```text
Role > Label > TestId > Text > CSS > XPath
```

## 🔁 Update File: `pages/LoginPage.ts`

---

### ❌ BEFORE (weak selectors)

```ts
usernameInput = '#user-name';
passwordInput = '#password';
loginButton = '#login-button';
```

---

### ✅ AFTER (enterprise locators)

```ts
import { Page, expect } from '@playwright/test';

export class LoginPage {
  constructor(private page: Page) {}

  usernameInput = this.page.getByPlaceholder('Username');
  passwordInput = this.page.getByPlaceholder('Password');
  loginButton = this.page.getByRole('button', { name: 'Login' });
  errorMessage = this.page.getByTestId('error');

  async goto() {
    await this.page.goto('https://www.saucedemo.com/');
  }

  async login(username: string, password: string) {
    await this.usernameInput.fill(username);
    await this.passwordInput.fill(password);
    await this.loginButton.click();
  }

  async validateLoginPageVisible() {
    await expect(this.loginButton).toBeVisible();
  }

  async validateLoginErrorVisible() {
    await expect(this.errorMessage).toBeVisible();
  }
}
```

# 9. ADVANCED LOCATOR FEATURES

---

## 🔹 Filter

```ts
await page.locator('div').filter({ hasText: 'Sauce Labs Backpack' });
```

---

## 🔹 First / Nth

```ts
await page.locator('button').first();
await page.locator('button').nth(2);
```

---

## 🔹 Parent-child relation

```ts
await page.locator('div', {
  has: page.getByText('Backpack')
});
```

---

# 10. AUTO-WAITING (CRITICAL)

---

Playwright automatically:

* waits for element to appear
* waits for element to be visible
* waits for it to be clickable

❌ DO NOT USE:

```ts
waitForTimeout(3000)
```

# 14. DEEP CONCEPTS

---

## Locator vs ElementHandle

* Locator → lazy (recommended)
* ElementHandle → static (avoid)

---

## Retry Mechanism

Locators automatically retry until timeout

---

## Stability Strategy

Bad:

```ts
div > div:nth-child(3) > button
```

Good:

```ts
getByRole('button', { name: 'Add to cart' })
```
