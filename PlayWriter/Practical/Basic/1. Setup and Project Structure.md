# Playwright Setup & Project Structure 

---

## 1. WHAT

Playwright is an end-to-end testing framework used to automate real browsers:

- Chromium
- Firefox
- WebKit

It helps us test a real web application by performing actions like:

- open website
- fill login form
- click buttons
- validate text
- validate URL
- validate cart and checkout behavior

For this complete practical, we are locking one website for all practice:

**Practice Website:** `https://www.saucedemo.com/`

---

## 2. WHY

Without Playwright:

- manual testing is slow
- repeated regression takes too much time
- cross-browser issues are missed
- checkout and login flows break silently

With Playwright:

- browser automation becomes fast
- validation becomes repeatable
- tests can run in parallel
- reports are generated automatically
- same test can run on multiple browsers

---

## 3. WHEN TO USE

Use Playwright when:

- you need UI automation
- you want browser-based end-to-end testing
- you need headless execution
- you want CI/CD-ready automation
- you want multi-browser testing
- you need screenshots, traces, videos, and reports

---

## 4. HOW

High-level flow:

```text
Test File → Playwright Runner → Browser Launch → Actions → Assertions → Report
```

Practical SauceDemo flow:

```text
Open SauceDemo → Enter username/password → Click Login → Validate Inventory Page
→ Add Product → Open Cart → Start Checkout → Fill Form → Finish Order → Validate Success Message
```

---

## 5. REAL-LIFE ANALOGY

Think of Playwright like a robot tester in a shopping mall.

- You = Test designer
- Playwright = Robot tester
- Browser = Shopping mall
- SauceDemo = Store
- Script = Instructions given to robot
- Assertion = Checking whether expected result happened
- Report = Final feedback from robot

Example:

You tell the robot:

1. Go to the store
2. Login with valid user
3. Add a bag to cart
4. Checkout
5. Confirm order completion

The robot repeats this exactly every time without forgetting steps.

---

## 6. ENGINEERING VIEW

Core components:

- **Test Runner** → executes test cases
- **Browser** → actual browser engine
- **Browser Context** → isolated user session
- **Page** → browser tab
- **Locator** → element finder
- **Action** → click, fill, type, select
- **Assertion** → validation
- **Report** → execution result
- **Page Object Model** → reusable business action layer

On SauceDemo:

- login page = authentication screen
- inventory page = products listing
- cart page = selected items
- checkout page = order form
- complete page = success confirmation

## File 3: `pages/LoginPage.ts`

```ts
import { Page, expect } from '@playwright/test';

export class LoginPage {
  constructor(private page: Page) {}

  usernameInput = '#user-name';
  passwordInput = '#password';
  loginButton = '#login-button';
  errorMessage = '[data-test="error"]';

  async goto() {
    await this.page.goto('https://www.saucedemo.com/');
  }

  async login(username: string, password: string) {
    await this.page.fill(this.usernameInput, username);
    await this.page.fill(this.passwordInput, password);
    await this.page.click(this.loginButton);
  }

  async validateLoginPageVisible() {
    await expect(this.page.locator(this.loginButton)).toBeVisible();
  }

  async validateLoginErrorVisible() {
    await expect(this.page.locator(this.errorMessage)).toBeVisible();
  }
}
```

Engineering note:

This page class handles only login page responsibilities.

---

## File 4: `pages/InventoryPage.ts`

```ts
import { Page, expect } from '@playwright/test';

export class InventoryPage {
  constructor(private page: Page) {}

  backpackAddButton = '#add-to-cart-sauce-labs-backpack';
  bikeLightAddButton = '#add-to-cart-sauce-labs-bike-light';
  cartLink = '.shopping_cart_link';
  title = '.title';

  async validateInventoryPageLoaded() {
    await expect(this.page).toHaveURL(/inventory/);
    await expect(this.page.locator(this.title)).toHaveText('Products');
  }

  async addBackpackToCart() {
    await this.page.click(this.backpackAddButton);
  }

  async addBikeLightToCart() {
    await this.page.click(this.bikeLightAddButton);
  }

  async openCart() {
    await this.page.click(this.cartLink);
  }
}
```

## File 6: `pages/CheckoutPage.ts`

```ts
import { Page, expect } from '@playwright/test';

export class CheckoutPage {
  constructor(private page: Page) {}

  firstName = '#first-name';
  lastName = '#last-name';
  postalCode = '#postal-code';
  continueButton = '#continue';
  finishButton = '#finish';
  successHeader = '.complete-header';

  async fillCheckoutInfo(first: string, last: string, zip: string) {
    await this.page.fill(this.firstName, first);
    await this.page.fill(this.lastName, last);
    await this.page.fill(this.postalCode, zip);

  async continueCheckout() {
    await this.page.click(this.continueButton);
  }

  async finishCheckout() {
    await this.page.click(this.finishButton);
  }

  async validateOrderSuccess() {
    await expect(this.page.locator(this.successHeader)).toHaveText('Thank you for your order!');
  }
}
```

```ts
import { test } from '@playwright/test';

test('valid login should open inventory page', async ({ page }) => {
```

```ts
import { LoginPage } from '../../pages/LoginPage';
```

```ts
import { CartPage } from '../../pages/CartPage';
import { CheckoutPage } from '../../pages/CheckoutPage';

  await checkoutPage.continueCheckout();
  await checkoutPage.finishCheckout();
  await checkoutPage.validateOrderSuccess();
```

# 12. PLAYWRIGHT CONFIGURATION

Create or update `playwright.config.ts`

```ts
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  fullyParallel: true,
  retries: 0,
  reporter: [['html'], ['list']],
  use: {
    headless: true,
    screenshot: 'only-on-failure',
    trace: 'on-first-retry',
    video: 'retain-on-failure',
  },
  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
    },
    {
      name: 'firefox',
      use: { ...devices['Desktop Firefox'] },
    },
    {
      name: 'webkit',
      use: { ...devices['Desktop Safari'] },
    },
  ],
});
```

What this gives:

- multi-browser execution
- screenshots on failure
- trace on retry
- video on failure
- HTML report

# 16. COMMON MISTAKES

## Mistake 1: Writing all selectors in test file

Problem:

- code becomes messy
- hard to maintain

Fix:

- move selectors to page classes

---

## Mistake 2: Hardcoding credentials in many files

Problem:

- duplicated data
- painful maintenance

Fix:

- keep credentials in `test-data/users.ts`

---

## Mistake 3: Not validating after action

Problem:

- click happened but no business confirmation

Fix:

- always add meaningful assertions

Example:

- after login validate URL or title
- after checkout validate success message

---

## Mistake 4: Using weak selectors

Problem:

- fragile automation

Fix:

- prefer stable selectors like ids, data-test, or strong CSS

---

## Mistake 5: Mixing business logic and validation everywhere

Problem:

- less readable tests

Fix:

- keep tests readable
- keep page classes reusable

# 17. INTERVIEW NOTES

### What is Playwright?
Playwright is a browser automation framework for end-to-end testing across Chromium, Firefox, and WebKit.

### Why is it strong?
It supports:
- built-in auto wait
- multiple browsers
- parallel execution
- screenshots, traces, videos
- modern test runner

### What is Browser Context?
An isolated session similar to a separate browser profile.

### Why use Page Object Model?
To separate test logic from UI interaction logic and improve reusability.

### Why SauceDemo for practice?
Because it provides:
- login flow
- product listing
- cart flow
- checkout flow
- negative login scenarios

### Why multi-browser testing matters?
Because same application can behave differently on Chrome, Firefox, and Safari/WebKit.
