# 🎭 Browser, Context, Page Concepts (Core Architecture)

---

## 1. WHAT

Playwright works using a **3-layer architecture**:

* **Browser** → actual browser engine
* **Browser Context** → isolated session
* **Page** → browser tab

Hierarchy:

```text
Browser → Context → Page
```

---

## 2. WHY

Without understanding this:

* tests share sessions ❌
* login state leaks ❌
* parallel execution breaks ❌

With this:

* isolation between tests ✅
* multi-user simulation ✅
* scalable automation ✅

---

## 3. WHEN TO USE

These concepts are used:

* in every Playwright test (internally)
* when testing multiple users
* when handling sessions (login/logout)
* when running parallel tests
* when managing cookies/storage

---

## 4. HOW

Execution flow:

```text
Test Runner → Browser → Context → Page → Actions → Assertions
```

---

# 8. MANUAL CONTROL (INCREMENTAL ADDITION)

---

So far, Playwright handled everything automatically.

Now we **explicitly control Browser → Context → Page**

---

## File: `tests/auth/multi-user.spec.ts` (NEW FILE)

```ts
import { test, chromium } from '@playwright/test';

test('multi-user session isolation', async () => {

  // 1. Launch browser
  const browser = await chromium.launch();

  // 2. Create two separate contexts
  const context1 = await browser.newContext();
  const context2 = await browser.newContext();

  // 3. Create pages inside each context
  const page1 = await context1.newPage();
  const page2 = await context2.newPage();

  // 4. Navigate both users
  await page1.goto('https://www.saucedemo.com/');
  await page2.goto('https://www.saucedemo.com/');

  // 5. Perform independent actions
  await page1.fill('#user-name', 'standard_user');
  await page2.fill('#user-name', 'locked_out_user');

  // 6. Close browser
  await browser.close();
});
```

# 13. COMMON MISTAKES

---

## Mistake 1: Using one context for all tests

Problem:

* session leakage

Fix:

* use separate contexts

---

## Mistake 2: Creating multiple browsers unnecessarily

Problem:

* slow execution

Fix:

* reuse browser, create contexts

---

## Mistake 3: Not closing browser

Problem:

* memory leaks

Fix:

* always close browser

---

## Mistake 4: Confusing page and context

Problem:

* incorrect architecture

Fix:

* remember hierarchy
