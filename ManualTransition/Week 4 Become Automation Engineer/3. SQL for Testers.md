# 4.3 SQL for Testers

## 🎯 What

SQL helps testers validate data directly in databases.

Focus:

* SELECT
* JOIN
* WHERE
* Validation use cases

Without SQL knowledge:

* weak backend validation
* overdependence on UI checks
* missed data defects

---

## ❓ Why This Matters in Automation

UI may show one thing.

Database may contain another.

SQL helps verify:

* inserted records
* updated data
* backend consistency
* integration correctness

Useful in automation and interviews.

---

## ⏰ When You Use SQL

Use when:

* validating test data
* checking database records
* verifying integrations
* supporting API and UI tests

# 🏗 SELECT

Used to retrieve data.

```sql id="k5v1qp"
SELECT * FROM users;
```

Gets all records.

---

## Select Specific Columns

```sql id="u3m9rd"
SELECT name,email
FROM users;
```

Use:

* targeted validation
* data checks

---

# WHERE

Filters data.

```sql id="q8r4lu"
SELECT *
FROM users
WHERE status='Active';
```

Use:

* record filtering
* expected data checks

---

## Numeric Filter

```sql id="f2m7tx"
SELECT *
FROM orders
WHERE amount>500;
```

---

# JOIN

Combines related tables.

```sql id="w6p3qa"
SELECT *
FROM orders o
JOIN customers c
ON o.customer_id=c.id;
```

Use:

* relational validation
* integration checks

---

## Why JOIN Matters

Applications often split data across tables.

JOIN validates relationships.

---

# Validation Use Cases

## User Record Validation

```sql id="d9m2rv"
SELECT *
FROM users
WHERE username='admin';
```

Check record created.

---

## API to DB Validation

API returns order id.

Validate in DB.

```sql id="x4r8pk"
SELECT *
FROM orders
WHERE id=101;
```

---

## Cross Table Validation

```sql id="t7m5lu"
JOIN orders
and customers
```

Verify relationship.
