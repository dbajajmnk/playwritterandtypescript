# 🏗 Synchronization Problems

## Typical Question

Element not clickable intermittently. What do you do?

Answer direction:

* avoid Thread.sleep
* use explicit waits
* wait for correct condition
* analyze timing root cause

Example:

```java id="k6m3qp"
wait.until(
ExpectedConditions
elementToBeClickable(locator));
```

---

## Common Discussion Points

* implicit vs explicit waits
* flaky timing issues
* dynamic loading handling

---

# Stale Element Handling

## Typical Question

How do you handle stale element exception?

Cause:

DOM changed after locating element.

---

## Strategy

Relocate element.

```java id="n4r9lu"
findElement again
```

Use retry if needed.

---

## Interview Points

Discuss:

* dynamic DOM refresh
* retry strategy
* robust locator strategy

# Parallel Execution Discussion

## Typical Question

Challenges in parallel execution?

Discuss:

* shared data conflicts
* thread safety
* driver isolation

---

## Key Point

Each thread should have separate driver.

Example concept:

```text id="r5m1qa"
Thread A → Driver A

Thread B → Driver B
```
