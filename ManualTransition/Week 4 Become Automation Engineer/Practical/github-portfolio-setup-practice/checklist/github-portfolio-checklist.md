# GitHub Portfolio Checklist

Before sharing your GitHub repository, check these items.

## Repository Setup

- [ ] Repository name is clear.
- [ ] Repository is public if you want to showcase it.
- [ ] README.md is available.
- [ ] `.gitignore` is available.
- [ ] Unwanted files are not uploaded.

## README Quality

- [ ] Project overview is clear.
- [ ] Tools used are mentioned.
- [ ] Test scenarios are explained.
- [ ] Run commands are included.
- [ ] Folder structure is shown.
- [ ] Interview talking points are included.

## Code Quality

- [ ] Folder structure is clean.
- [ ] Test files are organized.
- [ ] Config values are not hardcoded.
- [ ] Generated files are ignored.
- [ ] Code has meaningful comments.

## Interview Readiness

- [ ] You can explain project purpose.
- [ ] You can explain framework structure.
- [ ] You can explain how to run tests.
- [ ] You can explain future improvements.
