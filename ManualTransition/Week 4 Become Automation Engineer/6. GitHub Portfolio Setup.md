# 4.6 GitHub Portfolio Setup

## 🎯 What

GitHub portfolio setup means publishing your automation project in a clean and professional way.

Focus:

* Upload project
* Showcase framework

Without GitHub portfolio:

* project stays local
* hard to show work in interviews
* weak professional visibility

---

## ❓ Why This Matters in Automation

A good GitHub portfolio helps you:

* show practical work
* demonstrate framework thinking
* prove hands-on experience
* discuss project structure in interviews

It is not just code storage.

It is project presentation.

---

## ⏰ When You Use This

Use when:

* finishing automation project
* preparing for interviews
* building professional profile
* sharing framework work

Example:

```text id="g1p8mk"
Local Project
↓
GitHub Repository
↓
Public Showcase
```

# 🏗 Upload Project

## Step 1 Initialize Git

```bash id="w7m3qa"
git init
```

---

## Step 2 Add Files

```bash id="k5p9lu"
git add .
```

---

## Step 3 Commit Project

```bash id="u2m6rv"
git commit -m "Initial capstone project commit"
```

---

## Step 4 Create GitHub Repository

Create new repository on GitHub.

Example name:

```text id="p8r4mk"
selenium-restassured-capstone
```

---

## Step 5 Connect Local Project to GitHub

```bash id="d3n7tx"
git remote add origin <repository-url>
```

---

## Step 6 Push Code

```bash id="q6m1pw"
git push -u origin main
```
