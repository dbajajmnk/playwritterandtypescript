# 4.2 TestNG Foundations

## 🎯 What

TestNG is a Java testing framework used for structuring, executing and managing automated tests.

Focus:

* Test structure
* Assertions
* Parallel execution basics

Without a test framework:

* weak test organization
* no structured execution
* poor scalability

---

## ❓ Why This Matters in Automation

TestNG helps with:

* organizing tests
* assertions
* setup/teardown
* grouping tests
* parallel execution

It is common in Selenium frameworks.

---

## ⏰ When You Use TestNG

Use when:

* writing structured automation tests
* managing suites
* validating results
* running tests in parallel

# 🏗 Test Structure

## Basic Test Method

```java id="k4m9qx"
@Test
public void loginTest(){

System.out.println(
"Run Test");

}
```

Defines executable test.

---

## Setup Method

```java id="p7r2lu"
@BeforeMethod
setup();
```

Runs before each test.

---

## Teardown Method

```java id="c1v6ra"
@AfterMethod
cleanup();
```

Runs after test.

---

## Suite Structure

Tests can be grouped into suites.

Example:

```text id="z8m4dy"
Smoke Suite

Regression Suite
```

# Assertions

Assertions verify results.

## Basic Assertion

```java id="n5p8kw"
Assert.assertEquals(
actual,
expected);
```

---

## Boolean Assertion

```java id="r2m7tx"
Assert.assertTrue(
condition);
```

Use:

* result verification
* validations

---

## Common Mistake

Using prints instead of assertions.

# Parallel Execution Basics

Run multiple tests simultaneously.

Example concept:

```text id="w4r1qp"
Test 1 → Thread A

Test 2 → Thread B
```

---

## TestNG XML Example

```xml id="h6m3pv"

```

Enables parallel runs.
