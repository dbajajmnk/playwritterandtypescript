# 3.2 Playwright Basics

## 🎯 What

Playwright basics start with four core building blocks:

* Browser contexts
* Pages
* Locators
* Auto waiting

These are foundation concepts for writing tests.

Without understanding these:

* scripts feel confusing
* flaky automation increases
* architecture understanding stays weak

---

## ❓ Why This Matters in Automation

Most Playwright tests use these concepts constantly.

They help with:

* isolated sessions
* stable element handling
* clean test flow
* reduced synchronization issues

---

## ⏰ When You Use It

Use these in every Playwright script:

* opening isolated sessions
* creating tabs/pages
* locating elements
* waiting automatically for readiness

# 🏗 Browser Contexts

Context is isolated browser session.

```javascript id="p2d6ru"
const context=
await browser.newContext();
```

Use:

* separate users
* clean sessions
* isolated tests

Examples:

One context can act as User A.
Another as User B.

Common mistake:

Confusing browser with context.

---

# Pages

Page represents a browser tab.

```javascript id="v5r1qm"
const page=
await context.newPage();
```

---

## Open URL

```javascript id="n8m4zy"
await page.goto(
'https://example.com');
```

Use:

* navigation
* interactions
* validations

---

# Locators

Locators find elements.

```javascript id="j3q7pk"
page.locator(
'#username');
```

---

## Text Locator

```javascript id="u6x2ld"
page.locator(
'text=Login');
```

---

## Action with Locator

```javascript id="t1v5gc"
await page.locator(
'#login').click();
```

Use:

* stable element targeting
* readable tests

---

# Auto Waiting

Major Playwright strength.

Actions wait automatically.

```javascript id="w4m8ra"
await page.click(
'button');
```

Playwright waits for:

* visible
* attached
* actionable

No manual waits often needed.

---

## Auto Wait Example

```javascript id="k9p3ts"
await locator.fill(
'admin');
```

Wait built in.
