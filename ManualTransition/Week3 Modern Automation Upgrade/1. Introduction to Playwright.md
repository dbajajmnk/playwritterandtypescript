# 3.1 Introduction to Playwright

## 🎯 What

Playwright is a modern browser automation framework for web testing and automation.

Focus:

* Why modern frameworks
* Selenium vs Playwright
* Core concepts

It supports modern browsers:

* Google Chrome
* Mozilla Firefox
* Microsoft Edge

Without modern frameworks:

* more flaky automation
* slower feedback
* harder modern UI testing

---

## ❓ Why This Matters in Automation

Modern applications have:

* dynamic frontends
* SPAs
* async rendering
* API-heavy interactions

Modern frameworks help with:

* stability
* speed
* less flaky tests
* better developer experience

---

## ⏰ When You Use It

Use when:

* building modern automation stacks
* testing dynamic applications
* reducing Selenium complexity
* designing future-ready frameworks

# 🏗 Why Modern Frameworks

Modern frameworks solve:

* auto waiting
* parallel execution
* network interception
* browser isolation

Example strength:

Auto waits built in.

Less explicit wait handling.

---

# Selenium vs Playwright

## Selenium

Strengths:

* mature ecosystem
* broad adoption
* large community

Challenges:

* more wait handling
* driver dependency
* more framework plumbing

---

## Playwright

Strengths:

* auto waits
* fast execution
* isolated browser contexts
* built-in tracing

Example:

```javascript id="e4p9xt"
await page.click('button');
```

Wait handling built in.

---

## Comparison View

```text
Selenium
Driver Based

Playwright
Browser Engine Based
```

# Core Concepts

## Browser

Browser instance.

```javascript id="m6v2ra"
const browser=
await chromium.launch();
```

---

## Context

Isolated browser session.

```javascript id="z8k4qu"
const context=
await browser.newContext();
```

Like separate clean user profile.

---

## Page

Represents tab/page.

```javascript id="t3r7lf"
const page=
await context.newPage();
```

---

## Locator

Element targeting.

```javascript id="y1m5cp"
page.locator('#login');
```
