# 3.3 Assertions in Playwright

## 🎯 What

Assertions verify whether actual behavior matches expected behavior.

Focus:

* Validation patterns
* Reliable checks

Without assertions:

* actions may run
* but nothing is truly tested

Automation without assertions is incomplete.

---

## ❓ Why This Matters in Automation

Testing is validation.

Assertions check:

* element visibility
* text correctness
* page state
* expected outcomes

Assertions make automation:

* trustworthy
* reliable
* meaningful

---

## ⏰ When You Use Assertions

Use when:

* validating results
* checking UI behavior
* verifying business rules
* confirming test outcomes

Example:

```javascript id="g4m9ru"
await expect(page)
.toHaveTitle("Dashboard");
```

# 🏗 Validation Patterns

## Text Validation

```javascript id="u2n6pk"
await expect(locator)
.toHaveText("Success");
```

Checks exact text.

---

## Visibility Validation

```javascript id="d9r4lw"
await expect(locator)
.toBeVisible();
```

Checks element shown.

---

## URL Validation

```javascript id="x3m8qy"
await expect(page)
.toHaveURL(/dashboard/);
```

Validates navigation.

---

## State Validation

```javascript id="k6p1sv"
await expect(button)
.toBeEnabled();
```

Checks control state.

---

# Reliable Checks

## Prefer Web-First Assertions

Playwright assertions auto-wait.

```javascript id="w7t3ga"
await expect(locator)
.toBeVisible();
```

Wait built in.

Reliable.

---

## Avoid Manual Checks

Weak:

```javascript id="f4m9ku"
if(text=="Success")
```

Less reliable.

Prefer framework assertions.

---

## Use Stable Expectations

Validate:

* actual business outcome
* not cosmetic details only
