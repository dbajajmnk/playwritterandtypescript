# 3.5 REST Assured Introduction

## 🎯 What

REST Assured is a Java library used for API automation testing.

Focus:

* API automation concepts
* REST Assured setup
* Request and response testing
* Starter API tests

Without REST Assured:

* API testing in Java becomes more manual
* Request creation takes more boilerplate code
* Response validation becomes harder to organize

REST Assured helps testers write clean and readable API tests.

---

## ❓ Why This Matters in Automation

Modern applications are API driven.

REST Assured helps with:

* Backend validation
* Service-level testing
* Faster defect detection
* Java-based API automation
* Integration testing

In Selenium testing, we validate UI.

In REST Assured testing, we validate APIs directly.

---

## ⏰ When You Use REST Assured

Use REST Assured when:

* Testing REST APIs
* Validating status codes
* Validating JSON response fields
* Sending request payloads
* Testing GET, POST, PUT, PATCH, DELETE APIs
* Building Java API automation framework

Example:

```java
given()
.when()
    .get("/users")
.then()
    .statusCode(200);
```

# 🏗 API Automation Concepts

## Request

A request is what client sends to API.

Request contains:

* URL
* HTTP method
* Headers
* Query parameters
* Path parameters
* Body or payload

Example:

```http
GET /users/1
```

---

## Response

A response is what API returns.

Response contains:

* Status code
* Headers
* Body
* JSON data

Example:

```json
{
  "id": 1,
  "name": "Deepak",
  "role": "Tester"
}
```

---

## Assertion

Assertion verifies whether API response is correct.

Examples:

* Status code should be 200
* User name should be Deepak
* Response should contain id
* Error message should be correct

---

# 🔑 REST Assured BDD Style

REST Assured commonly follows this pattern:

```java
given()
.when()
.then();
```

## Meaning

```text
given()  → Request preparation
when()   → API action
then()   → Response validation
```

---

## Example

```java
given()
    .baseUri("https://api.example.com")
.when()
    .get("/users")
.then()
    .statusCode(200);
```

# 🌐 HTTP Methods In REST Assured

## GET

Used to fetch data.

```java
given()
.when()
    .get("/users")
.then()
    .statusCode(200);
```

---

## POST

Used to create data.

```java
given()
    .header("Content-Type", "application/json")
    .body("{\"name\":\"Deepak\"}")
.when()
    .post("/users")
.then()
    .statusCode(201);
```

---

## PUT

Used to update complete data.

```java
given()
    .header("Content-Type", "application/json")
    .body("{\"name\":\"Updated Deepak\"}")
.when()
    .put("/users/1")
.then()
    .statusCode(200);
```

---

## PATCH

Used to update partial data.

```java
given()
    .header("Content-Type", "application/json")
    .body("{\"role\":\"Admin\"}")
.when()
    .patch("/users/1")
.then()
    .statusCode(200);
```

---

## DELETE

Used to delete data.

```java
given()
.when()
    .delete("/users/1")
.then()
    .statusCode(204);
```

# 🧪 Starter Tests

## Test 1: Validate GET Status Code

```java
import org.junit.jupiter.api.Test;

import static io.restassured.RestAssured.*;
import static org.hamcrest.Matchers.*;

public class UserApiTest {

    @Test
    void getUsersShouldReturnStatus200() {
        given()
            .baseUri("https://jsonplaceholder.typicode.com")
        .when()
            .get("/users")
        .then()
            .statusCode(200);
    }
}
```

---

## Test 2: Validate JSON Field

```java
@Test
void getUserShouldReturnCorrectName() {
    given()
        .baseUri("https://jsonplaceholder.typicode.com")
    .when()
        .get("/users/1")
    .then()
        .statusCode(200)
        .body("name", equalTo("Leanne Graham"));
}
```

---

## Test 3: Validate Response Contains Field

```java
@Test
void getUserShouldContainEmailField() {
    given()
        .baseUri("https://jsonplaceholder.typicode.com")
    .when()
        .get("/users/1")
    .then()
        .statusCode(200)
        .body("email", notNullValue());
}
```

---

## Test 4: Validate POST API

```java
@Test
void createPostShouldReturnCreatedResponse() {
    String payload = """
        {
          "title": "REST Assured Introduction",
          "body": "Starter API automation test",
          "userId": 1
        }
        """;

    given()
        .baseUri("https://jsonplaceholder.typicode.com")
        .header("Content-Type", "application/json")
        .body(payload)
    .when()
        .post("/posts")
    .then()
        .statusCode(201)
        .body("title", equalTo("REST Assured Introduction"))
        .body("userId", equalTo(1));
}
```

# 📊 JSON Validation

REST Assured can validate JSON response using `body()`.

## Simple Field

```java
.body("name", equalTo("Deepak"));
```

---

## Nested Field

```java
.body("address.city", equalTo("Gwenborough"));
```

---

## Array Field

```java
.body("size()", greaterThan(0));
```

---

## First Item In Array

```java
.body("[0].name", notNullValue());
```

# 🧩 Request Data Handling

## Hardcoded Payload

Good for first practice only.

```java
.body("{\"name\":\"Deepak\"}")
```

---

## String Payload

Better readability.

```java
String payload = """
{
  "name": "Deepak",
  "role": "Tester"
}
""";
```

---

## Map Payload

Useful for dynamic values.

```java
Map<String, Object> payload = new HashMap<>();
payload.put("name", "Deepak");
payload.put("role", "Tester");
```

# 🔐 Headers

Headers provide request metadata.

Example:

```java
given()
    .header("Content-Type", "application/json")
.when()
    .post("/users")
.then()
    .statusCode(201);
```

Common headers:

* Content-Type
* Authorization
* Accept

---

# 🔎 Query Parameters

Query parameters filter or customize API response.

Example URL:

```http
/posts?userId=1
```

REST Assured example:

```java
given()
    .baseUri("https://jsonplaceholder.typicode.com")
    .queryParam("userId", 1)
.when()
    .get("/posts")
.then()
    .statusCode(200);
```

---

# 🛣️ Path Parameters

Path parameters are part of URL path.

Example URL:

```http
/users/1
```

REST Assured example:

```java
given()
    .baseUri("https://jsonplaceholder.typicode.com")
    .pathParam("id", 1)
.when()
    .get("/users/{id}")
.then()
    .statusCode(200)
    .body("id", equalTo(1));
```

---

# ⚠ Common Mistakes

## Mistake 1

Validating only status code.

Bad:

```java
.then()
.statusCode(200);
```

Better:

```java
.then()
.statusCode(200)
.body("name", notNullValue());
```

---

## Mistake 2

Hardcoding everything in test methods.

Avoid placing all URLs, payloads, and expected data directly inside every test.

---

## Mistake 3

Not checking negative scenarios.

Example:

```java
given()
.when()
    .get("/users/999999")
.then()
    .statusCode(404);
```

---

## Mistake 4

Not setting Content-Type for POST/PUT/PATCH.

Always set:

```java
.header("Content-Type", "application/json")
```

# 💻 Hands-On Lab Foundation Pattern

## Use Case 1: GET API Validation

Goal:

Validate user API response.

Steps:

```text
Set base URI
↓
Send GET request
↓
Validate 200 status
↓
Validate user id
↓
Validate name/email field
```

Code:

```java
@Test
void validateUserDetails() {
    given()
        .baseUri("https://jsonplaceholder.typicode.com")
        .pathParam("id", 1)
    .when()
        .get("/users/{id}")
    .then()
        .statusCode(200)
        .body("id", equalTo(1))
        .body("email", containsString("@"));
}
```

# 🎯 Interview Questions

## What is REST Assured?

REST Assured is a Java library used for testing REST APIs.

---

## Why use REST Assured?

It makes API automation readable, powerful, and easy to write in Java.

---

## What is given-when-then in REST Assured?

`given()` prepares request.

`when()` performs API action.

`then()` validates response.

---

## What is status code validation?

Checking whether API returned the expected HTTP status code.

---

## Why is JSON validation important?

Because status code alone does not confirm the correctness of response data.

---

## Difference between query parameter and path parameter?

Query parameter comes after `?`.

Path parameter is part of URL path.

Example:

```text
/users?id=1      → query parameter
/users/1         → path parameter
```
