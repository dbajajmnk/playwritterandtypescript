# 3.7 CI/CD Awareness

## 🎯 What

CI/CD means Continuous Integration and Continuous Delivery/Deployment.

Focus:

* Jenkins basics
* Pipeline overview
* Automation in delivery pipelines

CI/CD helps automate build, test and delivery flow.

Without CI/CD:

* manual deployments
* delayed feedback
* slower releases

---

## ❓ Why This Matters in Automation

Automation should run beyond local machines.

CI/CD helps:

* run tests automatically
* catch defects early
* support continuous delivery
* integrate testing into releases

---

## ⏰ When You Use CI/CD

Use when:

* running automated suites in pipelines
* triggering tests on code commits
* supporting deployments
* integrating QA into DevOps flow
