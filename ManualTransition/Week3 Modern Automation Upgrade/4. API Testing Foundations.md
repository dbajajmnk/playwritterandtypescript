# 3.4 API Testing Foundations

## 🎯 What

API testing validates application behavior at service level without UI.

Focus:

* HTTP methods
* Request/response basics
* Assertions
* JSON validations

Without API testing:

* only UI-level coverage
* slower defect detection
* weak backend validation

---

## ❓ Why This Matters in Automation

Many applications are API driven.

API testing helps:

* validate business logic faster
* test backend independently
* reduce UI dependency
* improve automation coverage

---

## ⏰ When You Use API Testing

Use when:

* testing backend services
* validating integrations
* checking responses
* automating service tests

# 🏗 HTTP Methods

## GET

Retrieve data.

```http
GET /users
```

Use:

* fetch records
* read data

---

## POST

Create data.

```http
POST /users
```

Use:

* create records
* submit payload

---

## PUT

Update data.

```http
PUT /users/1
```

---

## DELETE

Remove data.

```http
DELETE /users/1
```

---

# Request / Response Basics

## Request Contains

* URL
* headers
* body
* method

Example:

```json
{
"name":"Deepak"
}
```

---

## Response Contains

* status code
* headers
* body

Example:

```json
{
"id":1,
"name":"Deepak"
}
```

---

# Assertions

Validate response behavior.

## Status Assertion

```javascript id="g2m8qx"
expect(status)
.toBe(200);
```

---

## Field Assertion

```javascript id="t5r3lu"
expect(name)
.toBe("Deepak");
```

---

## Response Validation

Check:

* correct status
* correct payload
* correct business result

---

# JSON Validations

Validate JSON fields.

Example response:

```json
{
"user":"admin",
"active":true
}
```

---

## Field Validation

```javascript id="u8p4rv"
expect(response.user)
.toBe("admin");
```

---

## Nested JSON Validation

```javascript id="n1x7qd"
expect(response.data.id)
.toBe(1);
```
