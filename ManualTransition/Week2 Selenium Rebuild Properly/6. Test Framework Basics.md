# 2.6 Test Framework Basics

## 🎯 What

A test framework provides structure for organizing automation code.

Core parts:

* Base test setup
* Configuration handling
* Utilities

Without framework basics:

* repeated setup code
* hardcoded values
* poor reuse

---

## ❓ Why This Matters in Automation

As test suites grow, structure matters.

Framework basics help make automation:

* reusable
* maintainable
* scalable

Used in real projects and interviews.

---

## ⏰ When You Use This

Use when:

* building automation frameworks
* reducing duplicate setup code
* managing environments
* reusing common helpers

Example:

```java id="m6r2qa"
@BeforeMethod
setup();
```

# 🏗 Base Test Setup

Base test holds shared setup logic.

Example:

```java id="d5m1ly"
class BaseTest{

void setup(){}

void teardown(){}

}
```

Use:

* browser launch
* teardown
* shared initialization

---

## Setup Example

```java id="q8r6ga"
driver=
new ChromeDriver();
```

---

## Teardown Example

```java id="u3k9mn"
driver.quit();
```

Common mistake:

Repeating setup in every test.

---

# Configuration Handling

Store environment values outside code.

Examples:

* URLs
* browser name
* credentials
* timeouts

---

## Config Example

```java id="z2v7rp"
url=https://testapp.com
browser=chrome
```

Properties file style.

---

## Read Config

```java id="b4n5td"
config.getProperty(
"url");
```

Avoid hardcoding.
