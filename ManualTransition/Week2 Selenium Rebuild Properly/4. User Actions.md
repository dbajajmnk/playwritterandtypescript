# 2.4 User Actions

## 🎯 What

User Actions simulate real user interactions beyond simple click and type.

Examples:

* Mouse hover
* Right click
* Drag and drop
* Keyboard shortcuts

Without advanced actions:

* limited automation coverage
* incomplete UI testing
* missed user workflows

---

## ❓ Why This Matters in Automation

Many applications depend on complex interactions.

Examples:

* dropdown hover menus
* drag-drop widgets
* shortcut commands
* context menus

User actions make automation:

* realistic
* complete
* robust

---

## ⏰ When You Use User Actions

Use when:

* simple click does not work
* testing keyboard shortcuts
* handling hover menus
* automating drag and drop

Example:

```java id="d4n8qv"
Actions act=
new Actions(driver);
```

# 🏗 Mouse Interactions

## Hover

```java id="x5q1lz"
Actions act=
new Actions(driver);

act.moveToElement(menu)
.perform();
```

Use:

* hover menus
* tooltips

---

## Right Click

```java id="g9p4mn"
act.contextClick(element)
.perform();
```

---

## Double Click

```java id="k6t2ry"
act.doubleClick(button)
.perform();
```

---

## Drag and Drop

```java id="w3m8qc"
act.dragAndDrop(
source,target)
.perform();
```

Common mistake:

Element not visible before drag.

---

# Keyboard Interactions

## Send Keys

```java id="r7v5hd"
element.sendKeys("Hello");
```

---

## Keyboard Shortcut

```java id="n2z6pa"
act.keyDown(Keys.CONTROL)
.sendKeys("a")
.keyUp(Keys.CONTROL)
.perform();
```

Use:

* Ctrl+A
* Ctrl+C
* Ctrl+V

---

## Enter Key

```java id="j4q9lu"
element.sendKeys(
Keys.ENTER);
```

---

# Advanced Actions

## Action Chaining

```java id="c8m1tf"
act.moveToElement(menu)
.click(submenu)
.perform();
```

Multiple actions chained.

---

## Click and Hold

```java id="p6x3gw"

.perform();
```

---

## Release

