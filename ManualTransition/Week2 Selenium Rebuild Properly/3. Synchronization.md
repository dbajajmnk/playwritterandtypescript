# 2.3 Synchronization

## 🎯 What

Synchronization ensures automation waits correctly for application behavior.

Examples:

* Wait for page load
* Wait for button visible
* Wait for element clickable
* Handle slow responses

Without synchronization:

* flaky tests
* random failures
* timing issues

---

## ❓ Why This Matters in Automation

Applications do not always respond instantly.

Problems happen because:

* AJAX loads late
* APIs respond slowly
* elements render dynamically
* pages load in parts

Synchronization makes automation:

* stable
* reliable
* resilient

---

## ⏰ When You Use Synchronization

Use when:

* elements appear late
* pages load dynamically
* handling timing issues
* avoiding flaky scripts

Example:

```java id="n4q2zm"
wait.until(
ExpectedConditions
.visibilityOfElementLocated(locator));
```

# 🏗 Implicit Wait

Global wait applied to element searches.

```java id="g6u3pn"
driver.manage()
.timeouts()
implicitlyWait(
Duration.ofSeconds(10));
```

Use:

* basic synchronization
* simple projects

Common mistake:

Overusing implicit waits.

---

# Explicit Wait

Wait for specific condition.

```java id="m1v5qa"
WebDriverWait wait=
new WebDriverWait(
driver,
Duration.ofSeconds(10));

wait.until(
ExpectedConditions
.visibilityOfElementLocated(locator));
```

Use:

* better than hard waits
* condition-based waiting

---

## Clickable Wait

```java id="k5z9re"
ExpectedConditions
.elementToBeClickable(locator)
```

---

# Fluent Wait

Advanced wait with polling.

```java id="t4n8lu"
FluentWait<WebDriver> wait=
new FluentWait<>(driver)
.withTimeout(
Duration.ofSeconds(20))
.pollingEvery(
Duration.ofSeconds(2));
```

Use:

* dynamic systems
* polling scenarios

Common mistake:

Wrong polling configuration.

---

# Stability Strategies

## Prefer Explicit Waits

```text id="p9w4sj"
Avoid Thread.sleep()
Prefer Condition Waits
```

---

## Avoid Hard Waits

Bad:

```java id="h2c7yo"
Thread.sleep(5000);
```

Unstable.

---

## Wait For Correct State

Examples:

* visible
* clickable
* present

Not arbitrary delays.
