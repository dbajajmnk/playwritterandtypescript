# 2.5 Page Object Model

## 🎯 What

Page Object Model (POM) is a design pattern that separates page elements and page actions from test scripts.

Examples:

* Login page class
* Dashboard page class
* Reusable page methods
* Cleaner test structure

Without POM:

* locators inside tests
* duplicated code
* hard maintenance

---

## ❓ Why This Matters in Automation

As tests grow, scripts become messy.

POM helps by making automation:

* reusable
* maintainable
* scalable

Used heavily in Selenium frameworks.

---

## ⏰ When You Use POM

Use when:

* multiple test cases reuse pages
* framework design starts growing
* reducing duplicate locators
* organizing automation code

Example:

```java id="n6p2rv"
LoginPage login=
new LoginPage();
```

# Separation of Concerns

Keep responsibilities separate.

## Test Class

Only test flow.

```java id="p4g9yu"
login.login();

dashboard.verifyHome();
```

---

## Page Class

Contains:

* locators
* page methods
* page logic

Not assertions-heavy test logic.

---

## Bad Practice

```java id="r3d6kx"
Test file contains:

locators
clicks
validation
data
```

Everything mixed.
