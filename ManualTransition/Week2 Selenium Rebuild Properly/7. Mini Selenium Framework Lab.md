# 2.7 Mini Selenium Framework Lab

## 🎯 What

A mini Selenium framework lab combines the earlier Selenium topics into one starter structure.

Focus:

* base setup
* page objects
* locators
* config handling
* utilities
* test execution

Without framework structure:

* duplicate code
* poor maintainability
* weak project organization

---

## ❓ Why This Matters in Automation

Real projects do not keep everything in one file.

A starter framework helps:

* organize code properly
* reuse common logic
* separate responsibilities
* scale test cases

This is the first step from script writing to framework thinking.

---

## ⏰ When You Use This

Use when:

* moving from basic scripts to framework design
* creating reusable Selenium projects
* preparing for interviews
* starting real automation work

# 🏗 Starter Framework Structure

## Suggested Folder Structure

```text
project/
 ├── base/
 ├── pages/
 ├── tests/
 ├── utils/
 └── config/
```

---

## Base Layer

Contains:

* driver setup
* teardown
* common initialization

Example:

```java
class BaseTest{

void setup(){}

void teardown(){}

}
```

---

## Pages Layer

Contains:

* locators
* page methods

Example:

```java
class LoginPage{

void login(){}

}
```

---

## Tests Layer

Contains:

* test scenarios
* flow execution

Example:

```java
login.login();
dashboard.verifyHome();
```

---

## Utils Layer

Contains:

* screenshot methods
* waits
* helper methods

Example:

```java
takeScreenshot();
```

---

## Config Layer

Contains:

* URL
* browser
* timeout
* environment values

Example:

```text
browser=chrome
url=https://testapp.com
```
