# 2.2 Locators Deep Dive

## 🎯 What

Locators identify elements on a web page.

Examples:

* Find login button
* Locate username field
* Select menu item
* Capture dynamic elements

Without locators:

* no automation interaction
* no clicking
* no validations

---

## ❓ Why This Matters in Automation

Every Selenium script depends on locators.

Used for:

* click actions
* input fields
* assertions
* validations

Good locators make automation:

* stable
* maintainable
* reliable

---

## ⏰ When You Use Locators

Use when:

* finding web elements
* handling dynamic pages
* validating UI controls
* designing automation scripts

Example:

```java id="0n6vta"
driver.findElement(
By.id("username"));
```

# 🏗 ID Locator

Most preferred locator.

```java id="k6p1sx"
By.id("username")
```

Use:

* fast
* stable
* simple

Common mistake:

Using unstable generated ids.

---

# CSS Selectors

Uses CSS patterns.

```java id="z2m8ru"
By.cssSelector(
"#username")
```

---

## By Class

```java id="d5x4ga"
By.cssSelector(
".loginBtn")
```

---

## Attribute Selector

```java id="r8c1vq"
By.cssSelector(
"input[name='email']")
```

Use:

* flexible locating
* cleaner than XPath often

---

# XPath

Navigates element path.

```java id="j4f6tn"
By.xpath(
"//input[@id='user']")
```

---

## Text XPath

```java id="v9b3py"
By.xpath(
"//button[text()='Login']")
```

Use:

* complex locating
* text matching

Common mistake:

Overly long XPath.

---

# Dynamic Locators

Elements may change.

Use patterns.

```java id="q7n2ld"
By.xpath(
"//input[contains(@id,'user')]")
```

---

## Starts-With

```java id="p1w5sm"
//*[starts-with(@id,'menu')]
```

Handles dynamic ids.

---

# Locator Strategy Best Practices

## Prefer Order

```text id="f3r9ku"
ID
↓
CSS
↓
XPath
```

Preferred stability order.

---

## Use Unique Attributes

Good:

```java id="c5v8gx"
By.id("email")
```

Bad:

```java id="y2d7pl"
(//div)[6]
```

---

## Keep Locators Short

Readable and stable.
