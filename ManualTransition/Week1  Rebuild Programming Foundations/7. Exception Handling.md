# 1.7 Exception Handling

## 🎯 What

Exception handling manages runtime errors without crashing the program.

Examples:

* Element not found
* Timeout failure
* Invalid data issue
* Network/API error

Without exception handling:

* script stops abruptly
* fragile automation
* poor failure recovery

---

## ❓ Why This Matters in Automation

Failures happen in automation.

Examples:

* dynamic UI changes
* slow loading pages
* missing elements
* unstable environments

Exception handling makes automation:

* resilient
* recoverable
* maintainable

---

## ⏰ When You Use Exception Handling

Use when:

* handling failures gracefully
* protecting risky steps
* logging defects
* building retry logic

Example:

```java id="k4d2ph"
try{

login();

}
catch(Exception e){

retry();

}
```

# 🏗 try / catch / finally

## try

Code that may fail.

```java id="j7r2lu"
try{

clickButton();

}
```

---

## catch

Handles error.

```java id="c3n5fw"
catch(Exception e){

System.out.println(e);

}
```

---

## finally

Always executes.

```java id="u4v9kx"
finally{

driver.quit();

}
```

Use:

* cleanup
* browser closure
* resource release

# Error Handling Patterns

## Basic Catch Pattern

```java id="p5d7gh"
try{

login();

}
catch(Exception e){

System.out.println("Handled");

}
```

---

## Retry Pattern

```java id="n8f2cs"
try{

action();

}
catch(Exception e){

retry();

}
```

---

## Logging Pattern

```java id="q2k4wp"
catch(Exception e){

System.out.println(
e.getMessage());

}
```
