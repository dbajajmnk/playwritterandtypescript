# 1.4 Functions / Methods

## 🎯 What

Functions (methods in Java) are reusable blocks of code that perform a task.

Examples:

* Login action method
* Validation method
* Retry method
* Reusable test steps

Without methods:

* repeated code duplication
* hardcoded scripts
* poor maintainability

---

## ❓ Why This Matters in Automation

Automation is built from reusable steps.

Examples:

* login()
* logout()
* verifyTitle()
* createUser()

Methods make automation:

* reusable
* modular
* scalable

---

## ⏰ When You Use Methods

Use when:

* repeating same actions
* organizing test logic
* passing dynamic inputs
* returning results from validations

Example:

```java id="7lk9pt"
login();
verifyDashboard();
logout();
```

# 🏗 Method Creation

## Basic Method

```java id="t5q3fd"
void login(){

System.out.println("Login Executed");

}
```

Use:

* actions without return value

---

## Calling Method

```java id="4dr2zn"
login();
```

---

# Parameters

Input passed into method.

```java id="o9s7ah"
void login(String user,String password){

System.out.println(user);

}
```

Use:

* dynamic test data
* parameterized actions

Common mistake:

Hardcoding values inside method.

---

## Multiple Parameters

```java id="eh8r1m"
createUser(name,email,role);
```

---

# Return Values

Method can return data.

```java id="r1m7gf"
String getTitle(){

return "Dashboard";

}
```

---

## Boolean Return

```java id="n3g6vt"
boolean isLoginSuccess(){

return true;

}
```

Use:

* validations
* decision flows

---

# Reusable Test Methods

## Example

```java id="w5d2pl"
void verifyLogin(){

login();
verifyDashboard();

}
```

Reuse same method across tests.

# ⚠ Common Mistakes

## Mistake 1

Hardcoding instead of parameters.

Bad:

```java id="jx6kdn"
login("admin","123");
```

inside every test.

---

## Mistake 2

Very large methods.

Hard to maintain.

---

## Mistake 3

Ignoring return values.
