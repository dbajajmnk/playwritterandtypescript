# 1.1 Variables and Data Types

## 🎯 What

Variables are named containers used to store data in memory.

Examples:

* Store username
* Store password attempts
* Store expected test results

Without variables:

* no reusable automation logic
* hardcoded scripts
* poor maintainability

---

## ❓ Why This Matters in Automation

Every automation script depends on data:

* Test inputs
* Expected outputs
* Runtime state
* Dynamic locators
* API payload values

Variables make scripts:

* reusable
* configurable
* scalable

---

## ⏰ When You Use Variables in Real Testing

Use variables when:

* storing credentials
* validating titles
* capturing dynamic values
* handling response payloads
* parameterizing tests

Example:

```java
String username = "admin";
String expectedTitle = "Dashboard";
int retryCount = 3;
```

# 🏗 Types of Data

## Primitive Types

### int

Whole numbers

```java
int totalUsers=100;
```

Use cases:

* counts
* indexes
* retry attempts

Common mistake:
Using String instead of int.

---

## double

Decimal values

```java
double price=19.99;
```

Use:

* calculations
* metrics
* performance timing

---

## boolean

True or false.

```java
boolean loginPassed=true;
```

Use:

* validations
* conditions
* assertions

---

## char

Single character.

```java
char grade='A';
```

---

# Reference Types

## String

```java
String url="https://example.com";
```

Most used in automation.

---

# 🔒 Null Safety Thinking

Important concept:

```java
String token=null;
```

Null means value absent.

Common automation failures happen here.

Example:

```java
if(token!=null){
 validate(token);
}
```

# ⚠ Common Mistakes

## Mistake 1

Hardcoding everything

Bad:

```java
click("user123")
```

Better:

```java
String user="user123";
click(user);
```

---

## Mistake 2

Wrong type selection

Using:

```java
String count="5";
```

instead of

```java
int count=5;
```

---

## Mistake 3

Ignoring null handling

Very common defect source.
