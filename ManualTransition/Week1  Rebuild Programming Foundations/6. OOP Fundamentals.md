# 1.6 OOP Fundamentals

## 🎯 What

Object Oriented Programming organizes code into objects and reusable structures.

Examples:

* LoginPage class
* User object
* BaseTest parent class
* Reusable framework components

Without OOP:

* procedural scripts
* duplicate code
* weak framework design

---

## ❓ Why This Matters in Automation

Most automation frameworks are OOP-based.

Examples:

* Page Object Model
* Base test classes
* Utility classes
* Framework reuse

OOP makes automation:

* modular
* reusable
* maintainable

---

## ⏰ When You Use OOP

Use when:

* designing frameworks
* creating page objects
* sharing reusable logic
* modeling real systems

Example:


---

## Object

Instance of class.

```java id="r2d8tm"
User u1=

```

Use:

* modeling entities
* framework components

---

# Encapsulation

Binding data and methods together.

```java id="j5n3av"
class User{

private String name;

public void setName(String n){
name=n;
}

public String getName(){
return name;
}

}
```

Use:

* data protection
* controlled access

Common mistake:

Using public everywhere.

---

# Inheritance

Child class reuses parent behavior.

```java id="c9t7pk"
class BaseTest{

void launchBrowser(){}

}

class LoginTest
extends BaseTest{

}
```

Use:

* shared framework logic
* code reuse

---

# Polymorphism

Same method, different behavior.

```java id="w4m2zs"
class Login{

void execute(){
System.out.println("Standard");
}

}

class AdminLogin extends Login{

void execute(){
System.out.println("Admin");
}

}
```

Use:

* flexible design
* extensibility
