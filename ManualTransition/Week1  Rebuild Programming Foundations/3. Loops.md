# 1.3 Loops

## 🎯 What

Loops allow code to repeat a block of instructions multiple times.

Examples:

* Repeat action fixed number of times
* Iterate over data
* Retry failed operation
* Process collections

Without loops:

* repeated code duplication
* poor maintainability
* manual repetition

---

## ❓ Why This Matters in Automation

Automation constantly repeats operations:

* validating table rows
* checking list elements
* retrying failed steps
* processing test data

Loops make automation:

* reusable
* scalable
* efficient

---

## ⏰ When You Use Loops

Use when:

* repeating actions
* processing collections
* polling conditions
* implementing retries
* data-driven testing

Example:

```java id="skhhci"
for(int i=1;i<=3;i++){
retryLogin();
}
```

# 🏗 Types of Loops

## for Loop

Used when number of repetitions is known.

```java id="dpxgkc"
for(int i=1;i<=5;i++){
System.out.println(i);
}
```

Use:

* fixed iterations
* index processing
* retry counters

Common mistake:

Infinite loop through wrong condition.

---

## while Loop

Runs while condition remains true.

```java id="3d7r08"
while(elementNotVisible){
refresh();
}
```

Use:

* waiting logic
* polling conditions
* dynamic repetition

---

## Iteration Over Collections

```java id="w1w9pj"
String users[]={"A","B","C"};

for(String user:users){
System.out.println(user);
}
```

Use:

* test data loops
* list validations

# Retry Logic Concepts

## Basic Retry

```java id="zv9g6m"
for(int i=1;i<=3;i++){

if(login()){
break;
}

}
```

Retry up to 3 times.

---

## Polling Retry

```java id="fmr2t0"
while(!elementVisible){
checkAgain();
}
```

Used in automation synchronization.
