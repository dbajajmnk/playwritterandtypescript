# 1.5 Collections

## 🎯 What

Collections store multiple values together.

Examples:

* Multiple usernames
* Test data sets
* Product lists
* Table rows

Without collections:

* too many individual variables
* hardcoded data handling
* poor scalability

---

## ❓ Why This Matters in Automation

Automation often works with groups of data:

* multiple test users
* list validations
* data-driven testing
* API response collections

Collections make automation:

* organized
* scalable
* reusable

---

## ⏰ When You Use Collections

Use when:

* handling multiple values
* iterating data
* validating repeated UI elements
* managing test datasets

Example:

```java id="w3v7ra"
String users[]={"admin","test1","test2"};
```

# 🏗 Arrays

Fixed-size collection.

```java id="f8u3ya"
String users[]={"A","B","C"};
```

Use:

* fixed test data
* small datasets

---

## Access Array Elements

```java id="v2m8qo"
System.out.println(users[0]);
```

Index starts from zero.

Common mistake:

Accessing invalid index.

---

## Integer Array

```java id="q5j2lc"
int nums[]={10,20,30};
```

---

# Lists

Dynamic collections.

```java id="j9r6ph"
ArrayList<String> users=
new ArrayList<>();
```

---

## Add Values

```java id="n4c7ke"
users.add("Admin");
users.add("User");
```

---

## Get Values

```java id="m1w5tz"
users.get(0);
```

Use:

* dynamic datasets
* changing data sizes

---

# Iterating Collections

## Array Iteration

```java id="r8k2gb"
for(String user:users){

System.out.println(user);

}
```

---

## List Iteration

```java id="u0d6pj"
for(String item:list){

System.out.println(item);

}
```

Use:

* row validations
* data processing
