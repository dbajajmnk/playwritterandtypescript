# 1.2 Conditional Logic

## 🎯 What

Conditional logic allows a program to make decisions.

Examples:

* Execute one path if true
* Execute another path if false
* Control validation flow
* Handle alternate scenarios

Without conditions:

* no decision making
* static scripts
* poor failure handling

---

## ❓ Why This Matters in Automation

Automation constantly evaluates conditions:

* element visible or not
* login passed or failed
* response success or error
* expected vs actual result

Conditional logic makes automation:

* reusable
* resilient
* dynamic

---

## ⏰ When You Use Conditional Logic

Use when:

* validating outcomes
* handling alternate paths
* retry logic
* business rules
* response verification

```java id="t26m2q"
if(loginSuccess){
openDashboard();
}
else{
retryLogin();
}
```

# 🏗 Types of Conditional Logic

## if

```java id="7rqjtl"
if(elementVisible){
click();
}
```

Use:

* simple checks
* validations

Common mistake:
Missing braces in multi-line logic.

---

## if else

```java id="gxjcrx"
if(loginSuccess){
dashboard();
}
else{
showError();
}
```

Use:

* pass/fail decisions
* alternate flow

---

## Nested Conditions

```java id="2l0pgi"
if(userValid){
 if(passwordValid){
   login();
 }
}
```

Use:

* layered validations
* dependent checks

Common mistake:
Too much nesting.

---

## Switch Statements

```java id="y1j3of"
switch(role){

case "Admin":
manageUsers();
break;

case "User":
viewPortal();
break;

default:
denyAccess();

}
```

Use:

* fixed choices
* role logic

Common mistake:
Missing break.

